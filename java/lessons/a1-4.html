Limits of Finite Data Representation

Concept | Examples | Java Implementation

Getting Started: This short lesson will be primarily conceptual, covering the limits of different data types, helping you choose which data types are best to use when coding.

Intro: Common Conventions

Up until now, you have likely used int when working with integers despite the option of long and short. Additionally, with floating point numbers, you probably use double when there is also the option to use float. This lesson will clarify why these conventions exist.

Memory Limitations

When your machine executes a program, it has a limited amount of memory (RAM) available to use where needed in the program. Every variable declared takes up a certain amount of memory. Some data types take up more, while others take up less. The trade-off is that the types that take up less memory, in turn store a smaller number of digits. Here are some commonly used data types with their memory size and number bounds.

int - 4 bytes - -2,147,483,648 to 2,147,483,647
short - 2 bytes | -32,768 to 32,767
long - 8 bytes | -9,223,372,036,854,775,808 to 9,223,372,036,854,775,807

double - 8 bytes | 4.9E-324 to 1.7976931348623157E308
float - 4 bytes | 1.4E-45 to 3.4028235E38

boolean - 2 bytes
char - 2 bytes

Why Int and Double?

Firstly, int is used most frequently because it is right in the middle of short and long. It has the balance between taking up a medium amount of storage and a medium number of digits.

Secondly, double is used most frequently because it provides more accuracy than float. This is because of the way the floating point system works. To ensure the acurate storage of numbers, doubles are used more often. For example, in a banking application, it is very important for all numbers to be entirely accurate.

Of course, if you need to preserve memory, you can opt to use short and float. On the other hand, if you have plenty of memory and need to store large number and retain accuracy, you can use long and double.
