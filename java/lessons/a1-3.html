Non-Numeric Comparisons

Concept | Examples | Java Implementation

Getting Started: This lesson will cover how to compare non-numeric data types, such as strings, using the comparable interface.

Recall: Numercal Comparisons

Up to this point, you should be familiar with comparing numbers. For example 5>3 is true and 35<6 is false. The most common use for comparisons are if statements. Comparing numbers is simple; it can be done mentally with little to no issue. However,it may not be as obvious when it comes to strings and other non-numeric data types.

'Equal To' Comparisons (==)

Each respective language has it's own comparable interface. This lesson will be referring to Java's comparable interface. Please refer to your language's respective documentation (if you are not using Java) to find out what specific methods to use.

In Java, the easiest data type to compare is char since it is a primitive data type, unlike String, which is an object data type. Since it is a primitive data type, it can be compared following the same format as numerical data types like integers. For example, if ('C' == 'A') { System.out.println("They are equal!"); } works perfectly fine.

Strings are not primitive data types; they are objects (when declaring them, the first letter is capital, which indicates that it is an object). For this reason, using == to compare two Strings does not compare the String itself, but rather the reference of the String. It checks if the two String references you are comparing is referring to the same String object. If this sounds confusing at the moment - don't worry! The topic of Objects will be covered in a later lesson. For now, just know that for Strings, == cannot be used to compare them the way we compare primitive data types like ints, chars, etc.

The workaround then - for comparing objects - is to use the comparable interface. In Java, to see if two Strings are equal, the equals() method is used. The parameter for the method requires a String to be passed as an argument and the method is called using the other String that you want to compare it to. The method returns a boolean - true if the two Strings are equal to each other, false if they are not equal to each other. This replicates the == comparison operator that is used when working with primitive types.

Examples: 

String name1 = "Rob";
String name2 = "Dan";
String name3 = "Rob";
System.out.println(name1.equals(name2)); // false
System.out.println(name1.equals(name3)); // true
System.out.println("Anderson".equals("anderson")); // false (differences in upper/lower case will result in false)
System.out.println("Anderson".equals("Anderson")); // true

'Less/Greater Than' Comparisons (< and >)

Firstly, the char data type can use < and > just like other primitive data types. For example: 'A' < 'b' is a valid comparison. Every character has a corresponding integer value, which is what is actually compared in the background when comparing chars. 'B' is considered to be less than 'a' since capital letters come before lower case numbers - 'B' has a smaller corresponding integer valuie than 'a' does.

To compare Strings, the comparable interface has another method called compareTo(). This method is called using the same format as the equals() method. This method returns an integer, which can be negative, 0, or positive. To explain what the returned value means, lets say that String name1 = "Bob" and String name2 = "Janet" If we were to order these two names in alphabetical order, we know that "Bob" comes first. Likewise, if we called the compareTo method like so: name1.compareTo(name2); the number returned would be a negative. Therefore, we can conclude that when the String used to call the method (name1 in this example) is less than the String it is compared to (name2), a negative number is returned. On the other hand, if the comparison made was name2.compareTo(name1); a positive number would be returned, since name2 comes after name1, making name2 larger. Finally, if the two Strings being compared are equal, 0 is returned. This could technically be used to check if two Strings are equal, but the equals() method in the subsection above is easier to use.

Using these findings, we can make the following conclusions:

str1.compareTo(str2) > 0; // Checks if str1 is greater than (comes after) str2.
str1.compareTo(str2) >= 0; // Checks if str1 is greater than (comes after) or equal to str2.
str1.compareTo(str2) < 0; // Checks if str1 is less than (comes before) str1.
str1.compareTo(str2) <= 0; // Checks if str1 is less than (comes before) or equal to str1.

Java Implementation

This Java Implementation uses all the findings of this lesson in a simple program. 

public class A1_1_1 {

    public static void main(String[] args) {
        int a = 14;
        int b = 4;
        int c = a / b;
        System.out.println(c); // 3 (the quotient, using integer division, of 14/4 is 3).
        int d = a % b;
        System.out.println(d); // 2 (the remainder of 14 divided by 4 is 2)

    }

}
